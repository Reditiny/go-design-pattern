# 2.3 代理模式
由于某些原因需要给某对象提供一个代理以控制对该对象的访问。访问对象不适合或者不能直接引用目标对象，代理对象作为访问对象和目标对象之间的中介。

## 案例

+ 静态代理：火车站卖票。需要去到火车站（目标对象）买票，比较麻烦。火车站在多个地方都有代售点（代理对象），去代售点买票就方便很多了。
+ 动态代理：GO语言不适合实现动态代理

## 代理模式 优缺点

**优点：**

+ 代理模式在客户端与目标对象之间起到一个中介作用和保护目标对象的作用；
+ 代理对象可以扩展目标对象的功能；
+ 代理模式能将客户端与目标对象分离，在一定程度上降低了系统的耦合度；

**缺点：**

+ 增加了系统的复杂度；
